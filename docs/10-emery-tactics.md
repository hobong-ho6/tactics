# 실제 전술 분석 규칙 — 에메리의 아스톤 빌라

목적: 게임 구현의 원천이 되는 "실제 전술"을 세 층위로 분석·기록한다.
모든 결론은 근거(경기·평점·좌표)와 함께 DB에 남긴다. 문서는 방법론만 담고,
사실 데이터는 DB가 담는다.

## 분석 3층위

### 1. 팀 전술·철학 (에메리)
- 기록 대상: 기본 포메이션과 국면별 변형(빌드업 시 형태, 수비 블록, 압박 트리거),
  빌드업 경로, 공수 전환 원칙, 세트피스 경향.
- 팀 단위 사실은 **`tactic_observations`** 테이블에 기록한다
  (season, scope=philosophy/build_up/defence/in_possession/modulation/verdict, claim, evidence, source, confidence).
- 신규 분석 시: 반드시 특정 경기(들)나 실측 좌표를 근거로 지목하고 `matches`/`streaks`/`appearances`에 연결한다.

## 확립된 파운데이션 (25/26) — 근거는 tactic_observations 1–6

에메리는 스타일 이상주의자가 아니라 **균형·상대분석 우선의 실용주의자**다.
25/26의 구조적 상수(고정)와 경기별 가변축(모듈레이션)은 분리된다:

**고정 (파운데이션)**
- 공격 시 4-2-3-1 ↔ 수비 시 4-4-2 콤팩트 **미드블록** (10번이 왓킨스 옆으로 상승, 두 줄 4).
  강한 전방압박·초하이라인 팀이 아님 — 23/24의 "최고 하이라인" 서사는 25/26에 적용 금지.
- 마르티네스부터의 **숏 빌드업 고수** (강한 압박에도 유지, 탈출구는 왓킨스 롱볼).
- 공격은 **중앙 숫자·콤비네이션** 중심: 맥긴·로저스가 안으로, 왓킨스가 밖/뒷공간으로,
  풀백은 한 명씩만 전진.
- 인원 상수: 백4(캐시–콘사–토레스–디뉴/마첸), 카마라 앵커, 왓킨스 원톱.

**가변 (경기별 모듈레이션 4축)** — appearances 포지션 라벨 실측 근거
1. 10번의 정체: 로저스 중앙 vs 부엔디아 vs 틸레만스 프리롤
2. 로저스 LW 인버티드(기본) vs CAM 이동
3. 세컨드 피봇 프로필: 오나나(박스투박스) vs 틸레만스(DLP) vs 린델뢰프(비상 RDM)
4. 풀백 공격성: 캐시 수비적 vs 오버랩, 디뉴 vs 마첸

**판정: 파운데이션-퍼스트.** 게임 모델 자체를 경기마다 바꾸는 게 아니라,
고정 구조 위에서 인원·역할 스왑으로 색채를 조절한다.

## 게임 적용 정책 (파운데이션-퍼스트의 귀결)

- 팀 설정(`team_tactic_setups`)은 **kind가 달라도 원칙적으로 동일**해야 한다
  (빌드업 Short Passing / 수비 Balanced 미드블록 / 라인 ~52). 근거가 있을 때만 kind별로 다르게.
- 기본 전술 1개 = `kind='optimal'`. 경기별 변형은 `kind='match:<태그>'`로 쌓되,
  이는 "다른 철학"이 아니라 **역할·선발 스왑 프리셋**이다 (예: 로저스 CAM 변형).
- FC26에서는 기본 전술 + 변형 프리셋을 같은 팀 설정으로 저장해 경기 중 스위칭으로 재현한다.

### 2. 포지션별 요구 역할·역량
- 포지션마다 "에메리가 이 자리에 요구하는 것"을 정의한다. 근거는 그 자리에서
  높은 평점을 받은 선수들의 공통 행동(히트맵 존, 키패스, 수비 기여).
- 기록 위치: `player_role_map.rationale` 및 본 문서의 부록이 아니라 **DB**.
  포지션 요구사항을 요약할 필요가 생기면 `game_roles`와 대응시켜 기록한다.

### 3. 선수별 실측 분석
- 선수당 **베스트 경기**를 선정 (SofaScore API 평점 기준, 30분 이상 출전 경기):
  고정 포지션 선수는 **top-3**, 가변 포지션 선수는 **top-5** (포지션별 표본 확보 목적).
- 각 경기에 대해: 평점, 그 경기의 포지션·역할, 실측 히트맵(존 분포 + 중심좌표 + heat_map25),
  골/도움 → `appearances`에 기록 (rank_for_player 1..5).
- 랭킹·평점은 반드시 API 값 기준 — 뉴스 기반 평점으로 매긴 과거 랭크는 API로 교정한다.
- 최적 포지션 판정: 포지션별 평점 집계 + 팀 결과/연승 구간 가중 → `player_role_map`의
  `kind='optimal'` 행으로 기록, `rationale`에 数値 근거 명시.

## 선수별 임무·수행 관리 (`player_duties`)

포지션이 고정된 선수부터, 선수×시즌×포지션 단위로 다음 4단계를 기록한다:

1. **duties** — 에메리 체계가 그 자리에 요구하는 움직임 (근거: tactic_observations)
2. **execution** — 실측 히트맵·수치로 본 실제 수행 내용
3. **adherence** — 임무 대비 수행도 등급(HIGH/PARTIAL/LOW) + 요약.
   서사가 아닌 실측과 비교해서 판정한다.
4. **game_role_implication** — FC 역할/포커스 선택에 주는 결론
   (기존 player_role_map 매핑의 확정 또는 재검토 플래그)

용도: ① 선수 선발 자료 (같은 자리 경쟁자의 임무 수행 비교 — 예: 디뉴 vs 마첸),
② 게임 내 역할·포커스 정밀화. **주의 — top-3 표본 편향**: 베스트 경기는 대승에
치우치므로(피봇이 높게 올라가는 등), 결론이 편향에 민감한 선수는 어려운
경기/원정 표본을 추가 수집한 뒤 확정한다 (사례: 카마라).

**가변 포지션 선수**는 (player, position)별로 행을 분리한다. 포지션별 빈도·평점은
서사가 아니라 `player_match_positions`(경기별 평균위치 전수 수집) → `v_position_profile`
집계로 판정한다: 최다 출전 포지션(minutes)과 최고 수행 포지션(avg_rating, 표본≥3)을
구분해서 기록하고, 둘이 다르면 그 자체가 선발·매핑 판단 재료다
(사례: 맥긴 — 최다 RM 7.08 vs 최고 딥-라이트 7.30; 로저스 — 최다 CAM vs 최고 천장 LW).

## 증거 규칙

- 평점 권위: **SofaScore API 공식값 > SofaScore 뉴스 기사 > 서드파티 매체 자체 평점.**
  하위 소스로 기록했던 값을 API로 교정하면, 교정 이력을 `heat_summary`/`confidence`에 남긴다.
- 히트맵은 반드시 실측(API 좌표)으로 수집한다. 기사 서술만으로 heat_zones를 쓰지 않는다.
  수집 절차는 docs/30-data-rules.md.
- 서사 vs 실측 충돌: 실측 채택 + 충돌 기록 (CLAUDE.md 불변 규칙 3).
