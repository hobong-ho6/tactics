# 게임 내 검증 루프 — ⛔ 폐기 (2026-07-05)

**상태: 폐기.** 사용자가 수동 플레이(AI 자동 플레이 불가) 환경이라 체계적 검증이
불가능하다고 결정 — **데이터(실측 3축 + FC26 숙련도 타이브레이커)가 최종 심판**이다.
`team_tactic_setups.confidence`는 전부 `FINAL (data-only)`로 확정됐고, `ingame_checks`
테이블은 비운 채 보존한다(추가만 원칙). 아래 체크리스트는 참고 자료로만 남긴다 —
플레이 중 우연히 관찰한 것을 공유하면 기록은 가능하지만, 어떤 판정도 이를 전제하지 않는다.

---

(이하 원문 보존 — 참고용)

전술 재현의 최종 심판이었던 단계. 결과는 `ingame_checks` 테이블에 기록하고, 수렴하면
`team_tactic_setups`의 confidence를 HIGH로 올리고 `tactic_code`를 채우는 설계였다.

## 준비 (게임에서 1회)

1. 전술 2개 생성 — 팀 설정은 동일: **4-2-3-1 / 숏 패싱 (Short Passing) / 밸런스 (Balanced) / 라인 52**
   - A안 `(역할)`: 실측 최근접 조합 (재현 충실도) — 툴 [실측 → FC26 검증] 우측 패널 값
   - B안 `(최적)`: 처방 조합 — 툴 스쿼드 모드 왼쪽 패널 값
2. 각 전술의 **공유 코드**를 기록 → `team_tactic_setups.tactic_code` / `ingame_checks.tactic_code`

## 검증 플레이 (프리셋당 3~5경기)

경기 후 화면에서 확인할 것 — 3축 채점:

| 축 | 게임에서 보는 것 | 비교 기준 (DB) |
|---|---|---|
| 모양 shape | 선수별 히트맵 화면 (스크린샷) | `player_role_map` kind='measured'(+@dom/@tight/@poss) 그리드 |
| 기능 function | 경기 후 선수 스탯 (패스%·태클·슈팅·키패스) | `v_event_profile` (경기당 산출) |
| 도착 arrival | 득점·슈팅 위치와 유형 (헤더/6야드/에지) | `player_shot_profile` |

### 우선 관찰 포인트 (판정이 갈려 있는 자리)

- [ ] **부엔디아/로저스 10번** — 게임 히트맵이 좌편향으로 나오는가? (실측: 중심 y 56~64 좌측)
- [ ] **카마라 RDM (확정: 홀딩/볼위닝)** — "하프라인 우측-중앙 + 팀 듀얼 1위(8.4)·인터셉트(3.0)"가 재현되는가? (A/B는 숙련도 타이브레이커로 2026-07-05 종결)
- [ ] **오나나 LDM (확정: 홀딩/로밍)** — 오픈플레이 깊은 로머 무브먼트 + 볼위닝 기능 + 세트피스 헤더 도착이 나오는가? (A/B 종결, 동일)
- [ ] **GK 마르티네스** — 스위퍼/빌드업이 과하게 튀어나오지 않는가? (실측: 골라인 고정, 스위핑 상황부)
- [ ] **왓킨스** — 인박스 도착 비중 (실측 84%) + 헤더 득점 나오는가? 슛 위치 평균이 박스 안인가?
- [ ] **맥긴 RM** — 에지 도착(헤더 0, 6야드 거의 0)이 재현되는가, 아니면 박스로 파고드는가?
- [ ] **콘사 BPD** — 스텝아웃 캐리가 나오는가, 패스 성공률 90%+ 유지되는가?
- [ ] **팀 라인 높이 52** — 4-4-2 미드블록이 형성되는가? 너무 높거나 낮으면 ±5씩 조정 후 기록

## 기록 방법

경기(들) 끝나면 스크린샷과 함께 알려주면 세션에서 다음을 수행:
게임 히트맵 → 5×5 그리드 변환 → 실측과 코사인 비교, 스탯 대조, `ingame_checks` INSERT,
MISMATCH 항목은 역할/포커스 조정안 제시 → 다음 라운드.

## 종료 조건

- 우선 관찰 포인트가 전부 MATCH/PARTIAL 이상으로 수렴 →
  `team_tactic_setups.confidence='HIGH (in-game validated)'` + tactic_code 확정,
  해당 조합을 `player_role_map` kind='role'/'optimal'에 최종 반영.
- 수렴 실패 축이 있으면: 모양 불일치=역할 변경, 기능 불일치=포커스 변경,
  전체 형태 불일치=팀 설정(라인·빌드업) 변경 — 진단 분리 규칙.
